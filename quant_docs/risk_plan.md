# 风险计划
- 风险预算：以账户回撤分级控制为核心，不设硬止损（按你的策略约束）。
- 最大回撤：不做强制清仓；以仓位上限分级降杠杆。
- 杠杆/敞口限制：目标仓位范围 `[-1, 1]`，最大名义敞口约等于 1.0x 账户权益。

## 1. 风险触发与降级
### 1.1 分级回撤控制（与代码对齐）
- DD > 5%：最大仓位 0.8x
- DD > 10%：最大仓位 0.5x
- DD > 15%：最大仓位 0.2x（生存模式）

### 1.2 SAFE_MODE（自动降级）
- 触发：API 连续失败次数 >= 3（见 `execution_safety.py`）
- 行为：仅允许 reduce-only（减仓/平仓），禁止加仓/开仓
- 退出：人工干预（恢复后手动退出 SAFE_MODE）

### 1.3 人工停机与强平（手动）
- 停机：停止定时/后台进程，阻断后续下单。
- 强平：`live_trading_okx.py` 中提供 `force_close_all()`（reduce-only 市价全平）。

## 2. 风险备注
- 不设硬止损会放大尾部风险，需依赖分级降仓、SAFE_MODE 与监控告警协同。
- 实盘初期建议使用较小资金规模，且在稳定观察期内保留人工干预权。
